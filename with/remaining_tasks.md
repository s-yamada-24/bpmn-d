# 今後の課題と未実装機能

**作成日:** 2025-11-22
**ステータス:** 未着手

## 🚀 機能拡張

### 1. BPMN XML インポート
- **現状**: JSON形式のインポートのみ対応。BPMN XMLのエクスポートは可能だが、インポート機能がないため、他のBPMNツールで作成した図面を取り込めない。
- **課題**: BPMN 2.0 XMLパーサーの実装が必要。

### 2. Undo / Redo (元に戻す / やり直し)
- **現状**: 未実装。操作を誤った場合に復旧する手段がない。
- **課題**: コマンドパターンを用いた操作履歴管理システムの実装が必要。

### 3. 要素の整列・配置支援
- **現状**: グリッドスナップはあるが、要素同士の整列（左揃え、中央揃え等）や等間隔配置機能がない。
- **課題**: 複数選択時の整列コマンドの実装。

### 4. グループ化
- **現状**: 未実装。
- **課題**: 複数の要素をグループ化してまとめて移動・操作する機能の実装。

### 5. Swimlane (Pool/Lane) の機能強化
- **現状**: 基本的な作成とレーン追加・削除のみ。
- **課題**:
  - レーンの高さの自由なリサイズ
  - プール全体のサイズ変更時の内部要素の自動調整
  - ネストされたレーンのサポート

## 🛠️ 改善・最適化

### 1. パフォーマンス最適化
- **現状**: 要素数が増えた場合の動作検証が不十分。
- **課題**: 大規模な図面（数百要素）でのレンダリングパフォーマンスの確認と最適化。

### 2. エラーハンドリング強化
- **現状**: インポート時の不正なファイル形式に対するエラー処理が簡易的。
- **課題**: ユーザーフレンドリーなエラーメッセージとリカバリー処理の実装。

### 3. モバイル・タブレット対応
- **現状**: デスクトップ（マウス操作）前提のUI。
- **課題**: タッチ操作への対応、レスポンシブデザインの調整。

3. 要素の整列 (作図効率)
